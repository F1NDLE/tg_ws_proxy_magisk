# TG WS Proxy for Android (Magisk/KernelSU)

Инструмент для восстановления доступа к Telegram через технологию WebSocket. Реализован в виде автономного Magisk-модуля.


## Особенности
Автономность: В модуль встроен собственный интерпретатор Python 3.13 и необходимые библиотеки (cryptography, cffi и другие). Наличие Termux или системного Python не требуется.
Архитектура: Все компоненты изолированы внутри папки модуля, что исключает конфликты с системными файлами.
Управление: Возможность ручного переключения состояния прокси через функцию Action в менеджере модулей. Работает все бесперебойно!
Отладка: Система логирования процесса запуска и ошибок выполнения для диагностики проблем.

## Принципы работы и безопасность
Как это работает: Модуль запускает локальный SOCKS5-сервер, который упаковывает трафик Telegram в WebSocket-пакеты. Это позволяет обходить сетевые ограничения, так как трафик выглядит как обычное веб-соединение.
Локальный запуск: Прокси работает строго внутри устройства (127.0.0.1). Это значит, что ваши данные не проходят через сторонние публичные прокси-серверы, где они могли бы быть перехвачены.
Прозрачность: Весь исходный код скрипта находится в папке /lib/proxy. Вы можете лично убедиться в отсутствии скрытых функций или передачи данных на сторонние ресурсы.
Безопасность: Модуль не запрашивает доступ к вашим перепискам или ключам шифрования. Telegram использует сквозное шифрование, которое сохраняется и при работе через данный прокси.

## Установка
1. Скачайте архив с модулем.
2. Установите ZIP-архив через Magisk или KernelSU.
3. Перезагрузите устройство для инициализации системной службы.
4. Прокси будет запущен автоматически на порту 1080.

## Управление состоянием
В интерфейсе Magisk/KernelSU в меню модуля доступна кнопка Action:
Если прокси не активен — произойдет запуск.
Если прокси запущен — процесс будет завершен.
При ошибке запуска на экран будут выведены последние строки системного лога.

## Инструкция по настройке Telegram
Чтобы мессенджер начал работать через установленный модуль, выполните следующие шаги:
1. Откройте Telegram и перейдите в Настройки.
2. Выберите раздел Данные и память.
3. Прокрутите вниз и нажмите Настройки прокси.
4. Нажмите Добавить прокси и выберите тип SOCKS5.
5. Введите следующие данные:
   Сервер: 127.0.0.1
   Порт: 1080
   Логин: Можно оставить пустым, либо ввести любое значение.
   Пароль: Можно оставить пустым, либо ввести любое значение.
6. Сохраните настройки и включите переключатель "Использовать прокси".

## Структура модуля
/python — бинарные файлы и стандартная библиотека Python 3.13.
/lib — сторонние зависимости и исходный код прокси.
service.sh — сценарий автоматического запуска при загрузке системы.
action.sh — сценарий управления процессом (Toggle).
proxy_debug.log — журнал отладки (создается в директории модуля).
